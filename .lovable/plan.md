# Plano: PDF profissional e fiel ao preview da proposta\n\n## Diagnóstico atual\n\nHoje a geração do PDF está acontecendo no navegador, em `src/lib/pdf/generatePdf.ts`, com:\n\n- `html2canvas` para tirar um “print” do componente React renderizado fora da tela.\n- `jsPDF` para transformar essa imagem em PDF.\n- Template visual em `src/components/proposals/pdf/ProposalPdfTemplate.tsx`.\n- Preview da tela usando o mesmo `ProposalPdfTemplate`, mas exibido com `transform: scale(0.78)` dentro da interface.\n\nEsse método explica os problemas relatados:\n\n- `html2canvas` não renderiza igual ao Chromium real, principalmente fontes, baseline, gradientes, bordas, sombras e espaçamentos.\n- O PDF vira uma imagem dentro de um PDF, então a proporção depende do cálculo manual de canvas/página.\n- Medidas como `scrollHeight`, `getBoundingClientRect`, escala do canvas e conversão px/mm geram brancos, cortes e distorções.\n- Fontes carregadas via Google Fonts podem variar entre o preview e a captura.\n- Qualquer ajuste visual futuro pode quebrar novamente a exportação.\n\nConclusão: para PDF premium, o fluxo atual `html2canvas + jsPDF` deve deixar de ser o caminho principal do PDF.\n\n## Melhor tecnologia aplicável no ambiente atual\n\nA opção mais robusta para este projeto é gerar o PDF server-side em uma Supabase Edge Function usando um serviço de navegador headless já disponível no projeto: `BROWSERLESS_URL`.\n\nMotivo: Supabase Edge Functions não são um ambiente ideal para instalar e executar Chromium completo diretamente com Puppeteer/Playwright local.
